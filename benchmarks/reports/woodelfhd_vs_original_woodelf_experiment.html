How to Use This Report

This is an interactive benchmarking report comparing the runtime of different SHAP algorithms across a range of decision tree ensembles and dataset configurations. The sections below explain each part of the report.

Score. Each run compares all methods on the same task and parameters. The fastest method of each run scores 100; every other method scores (winner time / its time) × 100, so a method that took twice as long scores 50. Methods that crashed or are not supported for that task score 0. The overall score is the average across all runs.

Filtered score. The filter bar lets you narrow the score to a specific subset of runs by task type and parameter range. The key parameters are: n — the number of explained samples, m — the background dataset size (relevant for Background SHAP), and D — the maximum tree depth. The score updates instantly to reflect only the selected runs.

Experiment Analysis. The report contains many experiments. Each experiment changes one parameter while keeping the others fixed (for example, changing the tree depth D while keeping n and m fixed, or changing n while keeping D and m fixed). Use the Dataset and Mission dropdowns to choose an experiment. The plot will then show the runtime of each method in any value of the swept parameter. This makes it easy to compare methods and see how the parameter affects performance.

Open the Details: dataset, model & approaches panel below the controls to inspect the exact configuration of the selected experiment.

All Results. A full table of every individual run, including measured and extrapolated runtimes, the task type, and all parameter values. Times marked with * were extrapolated from runs with fewer trees and/or fewer samples.

Analytics. Summary statistics and method win-rate charts aggregated over all runs. Use the filters in this section to focus on a specific task type or a narrower parameter range.
