This report presents the experimental results from the paper “WOODELF-HD: Efficient Background SHAP for High-Depth Decision Trees” (arXiv link: ___). The paper introduces a new algorithm for computing Shapley values on decision tree models. As shown in this report, the method is significantly faster than existing approaches, especially for Background (Interventional) SHAP.

In these experiments we compare the runtime of several SHAP algorithms across different tree depths. For each dataset we trained LightGBM models with maximum depths 6, 9, 12, 15, 18, and 21. We then used each algorithm to compute Shapley values and Shapley interaction values using both the Path-Dependent and Background approaches. This report compares the running time of the different algorithms on these tasks.

Experiment Summary

SHAP Algorithms evaluated include the widely used shap package, PLTreeSHAP, FastTreeSHAP, the original Woodelf algorithm, and WoodelfHD.

Running Environment: All experiments were run on Google Colab using the CPU runtime with high memory enabled (50 GB memory). The notebooks used to run the experiments can be found here.

Parameters

Parameter Values Details
Max Depth D = 6, 9, 12, 15, 18, 21 We trained models with increasing tree depth. The graphs show how runtime changes as depth grows.
Number of trees T = 100 LightGBM models were trained with 100 trees. In some cases where running all trees was too expensive, results were computed on 10 trees or a single tree and then extrapolated.
Explained samples n = test set size SHAP values were computed on the entire test set. For SHAP interaction values we used n = 10,000 samples.
Background data size m = train set size For Background SHAP the entire training set was used as the background dataset. For the shap package we used m = 10 and extrapolated the runtime.
